## Claude Code Shell 探索 (18:40 - 18:53)

### 背景

用户注意到 Claude Code 渠道很重，为了避免风控，探索嵌入最新 Claude Code 的 WASM 方案。

### 分析过程

1. **方案探索**（5 种）:
   - ❌ 自行编译 WASM - Claude Code 闭源，不可行
   - ❌ SDK WASM 版本 - 不存在
   - ⚠️ 嵌入 JS 运行时 - 中等可行性
   - ✅ **真实 CLI 进程隔离** - 最佳方案
   - ⚠️ 请求模板复制 - 法律风险

2. **Claude Code CLI 分析**:
   - 下载并解压包: `@anthropic-ai/claude-code@2.1.87`
   - 大小: 30.4 MB (cli.js: 12.9 MB)
   - 提取网络层信息

3. **关键发现**:
   - ✅ **不需要签名！** 只需 `x-api-key` 认证
   - ✅ HTTP 头简单（5 个必需）
   - ✅ TLS 指纹可选
   - ✅ 可以实现轻量级壳子

### 关键决策

**不使用 WASM**，原因：
- Claude Code 无官方 WASM 版本
- 闭源项目，无法自行编译
- 法律风险高

**使用轻量级 Shell**，优势：
- 只提取网络层（TLS + HTTP 头）
- 无需内置提示词
- 简单认证机制（仅需 x-api-key）
- 易于维护和更新

## 重要决策记录

### 技术决策

1. **动态模型配置**
   - 决策: 不硬编码模型列表，使用数据库驱动
   - 原因: 灵活、易扩展、无需重启

2. **Claude Code 集成**
   - 决策: 使用轻量级 Shell（< 1MB）而非完整 CLI（100MB+）
   - 原因: 避免 CLI 的重量和内置提示词
   - 方案: 只提取 TLS 指纹 + HTTP 头模板

3. **WASM 嵌入**
   - 决策: 不使用 WASM
   - 原因: Claude Code 闭源，无官方 WASM 版本
   - 替代: 直接使用 HTTP 客户端 + 自定义 TLS

4. **TLS 指纹**
   - 决策: TLS 指纹为可选功能
   - 原因: 基本 API 调用不需要特殊 TLS
   - 后续: 可按需增强

   - 需要显式的转换函数

   - 需要统一使用一个 ORM 或分离连接池
