## 💡 技术决策

### sqlx 宏替换策略

**选择运行时版本而非离线数据的原因**：
1. 避免 `cargo sqlx prepare` 的循环依赖
2. 不需要维护 sqlx-data.json
3. 编译速度更快
4. 类型安全仍然由 Rust 保证

**缺点**：
- 失去编译时 SQL 语法检查
- 需要手动确保字段匹配

### OAuth Secrets 处理

**问题**：GitHub Push Protection 检测到硬编码的 secrets

**解决**：
1. 移除所有硬编码的 CLIENT_ID 和 SECRET
2. 使用环境变量配置
3. 创建新分支避免历史污染
4. 文档中说明配置方法

### 模型简化

**User 和 Account 模型**：
- 使用 String 存储枚举字段（role, status）
- 添加辅助方法 `role_enum()`, `status_enum()` 进行转换
- 避免复杂的 sqlx 类型映射
